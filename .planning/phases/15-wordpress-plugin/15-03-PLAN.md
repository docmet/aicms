## Objective

Build two frontend pieces:
1. A dashboard page (`/dashboard/wordpress`) where users register their WP site, view their MCP URL, and manage registered sites.
2. A public marketing page (`/wordpress`) with Matrix red/blue pill framing, feature list, and Stripe checkout link for the WP plugin subscription.

Purpose: Without the dashboard UI, users cannot register their WP site (WP-01, WP-02). Without the landing page, there is no acquisition channel (WP-07, WP-08).

Output: `frontend/src/app/dashboard/wordpress/page.tsx`, `frontend/src/app/(public)/wordpress/page.tsx`, shared component `RegisterWordPressSite.tsx`, API client `lib/api/wordpress.ts`.

<steps>
1. Create `frontend/src/lib/api/wordpress.ts`:
   - `registerWordPressSite(siteUrl: string, appUsername: string, appPassword: string): Promise<WordPressSite>`
     POST `/api/wordpress/sites` with auth header
   - `listWordPressSites(): Promise<WordPressSite[]>`
     GET `/api/wordpress/sites`
   - `deleteWordPressSite(siteId: string): Promise<void>`
     DELETE `/api/wordpress/sites/{siteId}`
   - TypeScript interface `WordPressSite { id: string; site_url: string; site_name: string | null; mcp_token: string; is_active: boolean; created_at: string }`
   - Use the same `apiFetch` or `fetch` pattern as other files in `frontend/src/lib/api/`

2. Create `frontend/src/components/wordpress/RegisterWordPressSite.tsx`:
   - Form with three fields: "WordPress Site URL" (url input), "Application Username" (text input), "Application Password" (password input — not the WP admin password, explain this)
   - Show helper text under Application Password: "Create this in WordPress Admin → Users → Profile → Application Passwords"
   - On submit: call `registerWordPressSite()`, show loading state, on success call `onSuccess(site)` callback prop, on error show toast via `useToast` from `@/hooks/use-toast`
   - Props: `onSuccess: (site: WordPressSite) => void`
   - Use shadcn/ui `Card`, `Input`, `Button`, `Label`

3. Create `frontend/src/app/dashboard/wordpress/page.tsx`:
   - Page title "WordPress Sites" with subtitle "Connect your WordPress sites to control content via AI chat"
   - If no sites registered: show `RegisterWordPressSite` form in a centered card
   - If sites registered: show a list of registered sites as cards, each card shows:
     - Site name (or site URL if name is null)

     - Delete button (with confirmation dialog using shadcn/ui `AlertDialog`)
   - Below the site list, show "Add another site" button that expands the `RegisterWordPressSite` form
   - Add nav link to this page: check `frontend/src/app/dashboard/layout.tsx` or nav component and add "WordPress" entry with a WP-like icon (use `Globe` from lucide-react)
