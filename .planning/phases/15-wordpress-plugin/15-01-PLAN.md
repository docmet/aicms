## Objective

Create the backend data layer for WordPress site registration and a HTTP client that proxies WP REST API calls. Users register their WP site (URL + Application Password) and MyStorey stores it. A thin service layer wraps WP REST API calls — used by MCP tools in plan 15-02.

Purpose: This is the auth + data foundation for the entire WP plugin feature. No WP MCP tools work without credential storage and the client service.

Output: `wordpress_sites` table, CRUD API at `/api/wordpress/sites`, and `WordPressClient` service class.

<task id="2">
<title>WP REST API client service and WordPress API router</title>
<steps>
1. Create `backend/src/services/wordpress_client.py`:
   - Class `WordPressClient(site_url: str, username: str, password: str)`
   - Method `async _request(method, path, **kwargs) -> dict | list`: Uses `httpx.AsyncClient` with `auth=httpx.BasicAuth(username, password)`. Base URL: `{site_url.rstrip("/")}/wp-json/wp/v2{path}`. Timeout 15s. Raises `HTTPException(502)` on connection error, re-raises `httpx.HTTPStatusError` so callers get the WP status code.
   - Method `async get_site_info() -> dict`: GET `/` on `{site_url}/wp-json/` (no `/wp/v2` prefix) → returns dict with `name`, `description`, `url`
   - Method `async list_posts(status="any", per_page=20) -> list`: GET `/posts?status={status}&per_page={per_page}&_embed=1`
   - Method `async get_post(post_id: int) -> dict`: GET `/posts/{post_id}?_embed=1`
   - Method `async create_post(title, content, status="draft", **extra) -> dict`: POST `/posts` with JSON `{title, content, status, **extra}`
   - Method `async update_post(post_id: int, **fields) -> dict`: PATCH `/posts/{post_id}` with JSON `fields`
   - Method `async list_pages(status="any", per_page=20) -> list`: GET `/pages?status={status}&per_page={per_page}&_embed=1`
   - Method `async get_page(page_id: int) -> dict`: GET `/pages/{page_id}?_embed=1`
   - Method `async create_page(title, content, status="draft", **extra) -> dict`: POST `/pages`
   - Method `async update_page(page_id: int, **fields) -> dict`: PATCH `/pages/{page_id}`
   - Method `async list_categories(per_page=50) -> list`: GET `/categories?per_page={per_page}`
   - Method `async list_tags(per_page=50) -> list`: GET `/tags?per_page={per_page}`
   - Method `async update_site_settings(title: str | None, description: str | None) -> dict`: POST to `{site_url}/wp-json/wp/v2/settings` with `{"title": title, "description": description}` (omit None values). WP REST API settings endpoint requires `manage_options` capability — Application Password auth provides this if the user is admin.

2. Create `backend/src/api/wordpress.py`:
   - FastAPI `APIRouter(prefix="/wordpress", tags=["wordpress"])`
   - `POST /sites` — register a new WP site:
     - Validate input with `WordPressSiteCreate`
     - Probe the WP site via `WordPressClient.get_site_info()` — if it fails, return 400 with `{"detail": "Cannot connect to WordPress site. Check URL and Application Password."}`
     - Generate `mcp_token = secrets.token_urlsafe(48)` — this is the token users will paste
     - Save `WordPressSite` to DB with `site_name` from WP info
     - Return `WordPressSiteResponse`
   - `GET /sites` — list user's WP sites (filter `is_active=True`, return all, no pagination needed for MVP)
   - `GET /sites/{site_id}` — get single WP site (verify ownership or 403)
   - `PATCH /sites/{site_id}` — update credentials/URL (re-probe WP if URL or password changed); generates new `mcp_token` only if explicitly requested or credentials change
   - `DELETE /sites/{site_id}` — soft-delete by setting `is_active=False`
   - All endpoints use `get_current_user` dependency from `src.api.auth`

3. Register the router in `backend/src/main.py`:
   - Add `from src.api import wordpress` import
   - Add `app.include_router(wordpress.router, prefix="/api")` alongside existing routers

4. Write `backend/src/tests/test_wordpress_api.py`:
   - Test `POST /api/wordpress/sites` with mock WP client (patch `WordPressClient.get_site_info` to return fake data)
   - Test `GET /api/wordpress/sites` returns list
   - Test `DELETE /api/wordpress/sites/{id}` sets `is_active=False`
   - Use existing test patterns from `backend/src/tests/` (AsyncClient, test DB)
</steps>
