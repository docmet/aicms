Add at the top of the file (with other stdlib/third-party imports):
```python
import sentry_sdk
```

After `settings = get_settings()` and before `async def _run_scheduled_publishes():`, add:
```python
# ── Sentry error monitoring ────────────────────────────────────────────────────
```

This placement ensures Sentry is initialized at module load time, before any request handling begins. The `[fastapi]` extra automatically captures unhandled exceptions from all route handlers.

**Step 2 — Configure Sentry alerting (new issues only, no spam):**

In the Sentry dashboard for the MyStorey project:
- Go to Alerts → Alert Rules
- The default rule fires on every new issue — this is correct behavior (first occurrence only)
- Disable or delete any "Send notification on every event" rules if they exist
- Set notification method to **email digest** (daily summary) rather than immediate per-error email, OR
- Connect to a dedicated Slack channel (e.g. `#mystorey-errors`) rather than direct messages

This step is manual (Sentry alert API requires org-level scoped token). Document the recommended settings in SUMMARY.md so they can be configured at project creation time.
