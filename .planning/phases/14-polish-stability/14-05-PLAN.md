Find the `_build_tools()` function in server.py (it returns a list of `Tool` objects). Enrich the following tools — leave all others untouched.

**update_page:**
Description: `"Update a page's title, URL slug, or published status. Confirm slug changes with the user first — changing a slug changes the public URL and may break existing links. To publish a page (make it live), use publish_page instead — it also creates a version snapshot. Use is_published: false to unpublish (take offline without deleting)."`

Add/update field descriptions:
- `site_id`: `"ID of the site containing the page (from list_sites or get_site_info)"`
- `page_id`: `"ID of the page to update (from list_pages or get_site_info)"`
- `title`: `"New page title, e.g. 'About Us'"`
- `slug`: `"New URL slug — lowercase letters and hyphens only, e.g. 'about-us'. WARNING: changing the slug changes the public URL and may break bookmarks or links."`
- `is_published`: `"Set false to unpublish (take offline). To publish, prefer using publish_page which also creates a version snapshot."`

**delete_page:**
Description: `"Permanently remove a page from a site (soft-delete). This cannot be undone from the MCP interface — the page and all its content sections are deleted. Always confirm with the user before calling this. If you want to hide a page temporarily, use update_page with is_published: false instead."`

**delete_site:**
Description: `"Permanently delete an entire site including all pages and sections (soft-delete). This cannot be undone. Always get explicit confirmation from the user before calling. Consider describing what will be lost (site name, number of pages)."`

**publish_page:**
Description: `"Publish a page's current draft content, making it live on the public site. Also creates a version snapshot (viewable with list_versions). Always publish after making content changes — draft changes are not visible to visitors until published. Returns a preview URL if available."`

**list_versions:**
Description: `"List the saved version history for a page (up to 5 most recent). Each version was created when the page was published. Use this before revert_to_version to pick the version number to restore."`

Field descriptions:
- `site_id`: `"ID of the site"`
- `page_id`: `"ID of the page to list versions for"`

**revert_to_version:**
Description: `"Restore a page's draft content to a previous published version. This overwrites the current draft but does NOT immediately publish — you must call publish_page after reverting if you want the restored content to go live. Use list_versions first to find the version_id."`

Field descriptions:
- `version_id`: `"ID of the version to restore to (from list_versions)"`

**update_site:**
If the current description is vague, update to: `"Update a site's display name, URL slug, or custom domain. Slug changes affect all page URLs under this site (/{site_slug}/{page_slug}). Confirm with the user before changing the slug."`

**generate_section:**
If missing field-level guidance, add to the `content_hints` or equivalent field: `"Optional JSON hints to guide content generation — e.g. {\"headline\": \"Coffeehouse on Main Street\", \"tone\": \"warm and inviting\"}. The tool will fill in the full section JSON from these hints."`
