<task type="auto">
  <name>Task 1: Add URL-paste tab to ImagePicker</name>
  <files>frontend/src/components/admin/ImagePicker.tsx</files>
  <action>
Redesign ImagePicker to support two input modes, toggled by a segmented control (two buttons: "Library" and "Paste URL"):

**Library mode (default):** Current behavior — clicking the empty state or "Change" opens the MediaLibrary modal.

**Paste URL mode:** Replace the library button with a text input. Validate with `new URL(value)` in a try/catch when the input loses focus (onBlur) or when Enter is pressed. If valid, call `onChange(value.trim())`. If invalid (URL constructor throws), show an inline error message below the input ("Enter a valid URL — e.g. https://example.com/image.jpg") and do NOT call onChange. Clearing the input should call `onChange(null)`.

Mode toggle state is internal (`useState<"library" | "url">("library")`). When a value is already set, show the image preview with "Change" (opens library) and "Remove" buttons regardless of mode — mode only affects the empty state input UX.

Use shadcn/ui `Button` for the two toggle buttons, styled as `variant="outline"` with `size="sm"`. Active mode button gets `variant="default"`. Place the toggle above the empty-state area (below the label, above the picker UI).

Import `Input` from `@/components/ui/input` for the URL input field.

Keep the `hint` prop behavior unchanged — shown below the component.

Do not import or use Tabs from shadcn — the two-button toggle is simpler and avoids an extra dependency.
